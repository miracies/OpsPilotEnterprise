# OpsPilot 项目情况总结

更新时间：2026-04-16  
范围：`opspilot-enterprise` 当前代码基线（基于仓库代码与路由实现盘点）

## 1. 项目定位

OpsPilot 是面向企业私有环境的 AIOps 平台，目标是打通以下闭环：

- 事件触发
- 自动分析
- 自动通知
- 人机协同处置（审批/执行）
- 自动归档复盘

当前已形成“前端控制台 + BFF + 多后端微服务 + 工具网关 + VMware/K8s 执行面”的分层架构。

## 2. 技术架构

### 2.1 总体分层

1. 前端层：`apps/web`（Next.js + React + TypeScript）
2. 接入聚合层：`apps/api-bff`（FastAPI）
3. 编排层：`services/langgraph-orchestrator`（对话/诊断/变更编排）
4. 工具控制层：`services/tool-gateway`（工具注册、路由、调用）
5. 领域能力层：
  - `services/vmware-skill-gateway`
  - `services/kubernetes-skill-gateway`
  - `services/change-impact-service`
  - `services/evidence-aggregator`
6. 运营与闭环层：
  - `services/event-ingestion-service`（事件、告警、审批、执行、审计、通知、案例）
  - `services/governance-service`（策略评估，已接 OPA）
  - `services/knowledge-service`
  - `services/approval-center-service`（保留）

### 2.2 核心技术栈

- 前端：Next.js 16、React 19、TypeScript、Tailwind CSS
- 后端：Python 3.12、FastAPI、Pydantic v2、httpx
- 数据存储：
  - Event Ingestion：SQLite（`events.db`）
  - Secrets：SQLite（BFF 密钥存储）
  - Chat 会话：当前主要为 BFF 进程内内存状态
- 模型能力：OpenAI 兼容接口（默认 GLM），支持降级/兜底
- 策略引擎：OPA（Docker 部署，Governance 调用）

### 2.3 部署架构

`deploy/docker/docker-compose.yml` 已包含以下关键容器：

- `web`、`api-bff`、`orchestrator`、`tool-gateway`
- `vmware-gateway`、`kubernetes-gateway`
- `change-impact`、`evidence-aggregator`
- `event-ingestion`、`governance`、`knowledge`、`approval-center`
- `opa`（8181，挂载 `policy/rego`）

## 3. 功能架构

当前功能可分为 8 个域：

1. AI 对话与智能编排
2. 资源中心（vCenter / K8s）
3. 事件中心与自动分析
4. 执行申请与自动/审批执行
5. 审批中心与通知中心
6. 变更影响分析
7. 审计中心与案例复盘
8. 连接与密钥治理、工具治理、策略治理

## 4. 功能实现情况（按模块）

### 4.1 AI 对话与编排

- 状态：**已实现（核心可用）**
- 已落地能力：
  - 会话与消息接口、异步 in-progress 消息
  - 执行状态时间线（progress events）
  - `reasoning_summary` 结构化摘要
  - 资源查询意图（vCenter 生产环境 VM 数量）
  - 导出意图（VM 列表导出 + 对话下载链接）
  - 按用户指定列导出（含忽略列提示）
  - 通用运维问答 `generic_ops_qa`（知识检索 + 模板回答）
  - VM 电源相关意图路由（避免落入通用闲聊）
- 现状说明：
  - 已有 Orchestrator 主路径 + BFF fallback 双路径
  - 部分场景仍保留 fallback 分支（但已显式避免静默 mock）

### 4.2 资源中心（vCenter / K8s）

- 状态：**已实现（真实连接主路径可用）**
- 已落地能力：
  - vCenter 概览/清单查询
  - K8s 概览/工作负载查询
  - VM 清单 CSV 导出（10 分钟下载）
  - 导出富字段拼装（IP、Host、CPU、内存、存储、datastore）
- 现状说明：
  - BFF 通过 `connections + secrets` 组装连接上下文
  - 存在 Tool Gateway / VMware Gateway 回退链路（非 mock 数据回退）

### 4.3 事件中心与自动分析闭环

- 状态：**已实现（主闭环可用）**
- 已落地能力（Event Ingestion）：
  - 监控控制：`/monitoring/status|start|stop`
  - 事件接入与 Incident 生成
  - 多轮分析循环（预算：5 轮 / 60 秒，含停止条件）
  - 分析过程结构化记录（`analysis_process`）
  - 用户级自动处置偏好（`suggest_only|low_risk_auto|full_auto`）
  - 状态变化触发复分析 + 手动复分析
- 现状说明：
  - 规则与动作白名单已存在
  - 监控默认 60 秒周期

### 4.4 执行申请与处置执行

- 状态：**已实现（端到端主流程可用）**
- 已落地能力：
  - `executions` API：dry-run / submit / detail / list / cancel
  - SQLite 持久化：`execution_requests`、`execution_targets`、`execution_steps`
  - 策略联动：根据 policy 结果区分自动执行或待审批
  - 动作能力矩阵（single/batch）
  - 前端执行申请页：动作表单、风险展示、目标对象、dry-run、审批流、提交按钮

### 4.5 审批中心与通知中心

- 状态：**已实现（真实数据联动）**
- 已落地能力：
  - 审批列表、审批详情、审批决策（通过/驳回）
  - 驳回意见前端必填校验
  - 5 秒轮询刷新
  - 通知列表、ack、值班信息展示
- 现状说明：
  - BFF 已代理到 Event Ingestion 真实接口

### 4.6 变更影响分析

- 状态：**已实现（可用）**
- 已落地能力：
  - `change-impact/analyze` 实际调用 Tool Gateway + Governance
  - 支持 VMware/K8s 目标分析
  - 输出风险分、风险等级、影响对象、检查项、回退建议、拓扑依赖
  - 已包含 `vmware.host_restart` 对应动作识别

### 4.7 审计、知识、案例

- 审计与案例：
  - 状态：**已实现（真实主链）**
  - 由 Event Ingestion 持久化与查询（审计日志、案例列表/详情）
- 知识库：
  - 状态：**已实现（检索可用）**
  - 用于通用问答证据引用（`evidence_refs`）

### 4.8 连接中心、认证、治理域

- 连接中心：
  - 状态：**部分实现**
  - 说明：当前仍为 `MOCK_CONNECTIONS` 内存模型，但可绑定真实 endpoint 与 secret 引用
- 认证：
  - 状态：**部分实现**
  - 说明：`auth` 仍使用 `MOCK_USERS`
- Governance：
  - 状态：**部分实现**
  - 说明：
    - `policies/evaluate` 已接 OPA（真实策略判定）
    - `audit/policies/upgrades` 仍存在内存示例数据
- Evidence Aggregator：
  - 状态：**部分实现**
  - 说明：当前以 mock 聚合数据为主，尚未完全替换为真实多源证据汇聚

## 5. “真实数据化”现状结论

已基本完成真实化的关键主链：

- vCenter / K8s 资源查询与执行
- 事件 -> 分析 -> 通知 -> 审批/执行 -> 归档
- 审批中心与执行申请联动
- OPA 策略外部化接入

仍需继续去 mock 的重点：

1. 连接中心（连接元数据仍以内存 mock 为主）
2. 认证用户体系（仍为 mock 用户）
3. Evidence Aggregator（仍有 mock 证据生成）
4. Governance 部分列表接口（策略/审计列表仍有示例数据）

## 6. 当前版本总体评估

- 架构成熟度：**高**
- 核心闭环可用性：**中高**
- 生产就绪度：**中**
- 主要短板：**治理域和基础域仍有局部 mock，需进一步持久化与统一身份体系**
