# 一、统一总提示词（先给 Codex）

先把这段作为项目级总提示词，建议在做任何页面前先执行一次。

```text
你现在是资深产品设计师、企业级前端架构师、AIOps 平台 UX 设计师。

请为一个企业级 AIOps 平台设计并实现完整的 Web 前端界面。

一、技术栈要求
- Next.js
- React
- TypeScript
- Tailwind CSS
- shadcn/ui
- lucide-react
- recharts
- 必要时可使用 framer-motion，但动画要克制

二、平台定位
这是一个企业内部使用的 AIOps 平台，主要服务于：
- 运维工程师
- 平台管理员
- 值班人员
- 审批人员
- 安全审计人员
- IT 管理者

三、整体设计目标
- 企业级、中后台风格
- 专业、稳定、现代，但不要花哨
- 可长时间使用，不造成视觉疲劳
- 信息层级清晰
- 支持复杂业务，但界面不能混乱
- 强调“分析过程可见、证据可见、风险可见、操作可审计”

四、视觉风格要求
- 主风格：蓝白商务风，简洁克制
- 支持亮色模式，结构上预留深色模式扩展能力
- 不使用夸张渐变
- 不使用过重阴影
- 圆角中等偏小，体现企业级稳重感
- 图标简洁统一
- 页面留白适中
- 表格、卡片、状态标签、筛选器风格统一

五、设计系统要求
先建立统一 design system，包括：
1. 颜色体系
   - primary / secondary / success / warning / danger / neutral / info
2. 字体层级
   - page title / section title / card title / body / helper / mono
3. 间距体系
   - 4 / 8 / 12 / 16 / 20 / 24 / 32
4. 圆角体系
   - input / button / card / modal
5. 阴影体系
   - sm / md，仅少量使用
6. 状态体系
   - 正常、告警、严重、处理中、已关闭、待审批、已拒绝、执行中、失败、成功
7. 通用组件
   - AppShell
   - SideNav
   - TopBar
   - Breadcrumb
   - PageHeader
   - SectionCard
   - FilterBar
   - SearchInput
   - StatusBadge
   - SeverityBadge
   - RiskBadge
   - AIConfidenceBadge
   - Timeline
   - EmptyState
   - LoadingSkeleton
   - DetailDrawer
   - SplitPanel
   - ResizablePanel
   - JsonCodeViewer
   - LogViewer
   - EvidenceList
   - AuditTrail
   - DiffViewer
   - ChatMessage
   - AgentFlowGraph

六、布局规范
- 左侧为主导航
- 顶部为全局状态栏，展示搜索、环境、告警数、用户菜单
- 内容区域采用统一的 page container
- 支持 dashboard / detail / workspace / form / table 五类布局模板
- 1440 宽度优先设计，同时兼顾 1280 和 1024
- 小屏下侧边栏可折叠
- 详情页尽量支持左右分栏或上下可调整布局

七、交互规范
- 高风险操作要明显二次确认
- AI 分析结果必须展示证据来源、置信度、时间戳
- 所有自动执行建议要展示风险等级和审批状态
- 日志、事件、证据、变更 diff 要支持筛选和跳转联动
- 表格需要支持：
  - 搜索
  - 高级筛选
  - 列显示控制
  - 分页
  - 行级详情
- 抽屉优先于新页面处理轻量详情
- 复杂分析优先工作台布局而不是弹窗

八、代码要求
- 目录结构清晰
- 组件高度复用
- 不要把所有页面写成巨型文件
- 先使用 mock data
- 数据模型和 UI 分层清楚
- 保留后续接入真实 API 的结构
- 所有页面使用统一的页面骨架和设计语言

九、输出顺序
请按以下顺序输出：
1. 项目目录结构建议
2. design tokens
3. 全局 layout
4. 通用组件
5. 页面模板
6. 再逐个实现业务页面

十、验收标准
- 第一眼专业，不像 demo
- 页面间风格统一
- 页面信息密度适合运维场景
- 重要状态、风险、审批信息一眼可见
- AI 相关能力不是“聊天框摆设”，而是可解释、可追踪、可审计
```
